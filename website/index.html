// what it catches

30 checks · 10 categories

Every check targets a vulnerability pattern AI code generators introduce. Each includes detection commands and fix instructions.

ASecrets & Credentials
  • API keys in frontend code
  • .env files in git history
  • Weak JWT secrets
BAuth & Sessions
  • No rate limiting on auth
  • JWTs in localStorage
  • Tokens that never expire
  • Sessions not invalidated
  • Weak password hashing
CAuthorisation
  • Frontend-only admin auth
  • Missing auth middleware
  • IDOR vulnerabilities
DInjection & Input
  • SQL injection
  • XSS via unsafe rendering
  • Missing input validation
  • eval / exec / pickle
EInfrastructure
  • CORS wildcard
  • Running as root
  • Database port exposed
  • No HTTPS enforcement
  • Debug mode in production
FError Handling
  • Stack traces in responses
GBaaS & Third-Party
  • Supabase/Firebase RLS off
  • Unsigned payment webhooks
  • Hallucinated dependencies
HFile Handling
  • Uploads without MIME check
ITransport & Headers
  • Open redirects
  • Missing security headers
  • CSP missing or permissive
JOps Security
  • No logging or monitoring
  • No backup or recovery
