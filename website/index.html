// the problem

LLMs optimise for "works."
They stop before "secure."

AI code generators build fast. They also hardcode API keys, skip auth middleware, disable Row Level Security, and hash passwords with MD5. Every time.

01
Secrets in your frontend
Stripe secret keys in React components. Supabase service role keys exposed via public endpoints. The model picks the shortest path to working code — through your credentials.
02
Auth that looks right but isn't
No rate limiting on login. JWTs in localStorage. Tokens that never expire. Passwords hashed with MD5. The login page works. The security behind it doesn't.
03
Permissions checked on the wrong side
Admin logic in the frontend. Auth middleware on the first three routes but not the rest. Change the user ID in the URL and you own someone else's data.
04
Injection bugs solved 20 years ago
SQL injection via string concatenation. XSS through dangerouslySetInnerHTML. eval() running user input. The shortest path to working code skips input validation.
05
Everything around the app misconfigured
CORS wildcard. Debug mode in production. Database ports exposed. Supabase RLS disabled — the same config that caused the Moltbook breach. 1.5M API keys leaked.

// what it catches

30 checks · 10 categories

Every check targets a vulnerability pattern AI code generators introduce. Each includes detection commands and fix instructions.

ASecrets & Credentials
  • API keys in frontend code
  • .env files in git history
  • Weak JWT secrets
BAuth & Sessions
  • No rate limiting on auth
  • JWTs in localStorage
  • Tokens that never expire
  • Sessions not invalidated
  • Weak password hashing
CAuthorisation
  • Frontend-only admin auth
  • Missing auth middleware
  • IDOR vulnerabilities
DInjection & Input
  • SQL injection
  • XSS via unsafe rendering
  • Missing input validation
  • eval / exec / pickle
EInfrastructure
  • CORS wildcard
  • Running as root
  • Database port exposed
  • No HTTPS enforcement
  • Debug mode in production
FError Handling
  • Stack traces in responses
GBaaS & Third-Party
  • Supabase/Firebase RLS off
  • Unsigned payment webhooks
  • Hallucinated dependencies
HFile Handling
  • Uploads without MIME check
ITransport & Headers
  • Open redirects
  • Missing security headers
  • CSP missing or permissive
JOps Security
  • No logging or monitoring
  • No backup or recovery
